Projeto web: tecnologias, tendências, gestão de projetos (custos, calendários de tarefas, relatórios de acompanhamento)

Introdução

Este material é voltado para projetos web, começando pela definição de projeto: um esforço temporário destinado a alcançar um objetivo claro e viável, com prazos definidos e recursos limitados, como financeiros, humanos e materiais. Os projetos web geralmente envolvem iniciativas novas ou exclusivas, que, muitas vezes, não se repetem.

Na elaboração de um projeto para sistemas web, é fundamental definir e planejar todas as informações necessárias, para que o produto a ser entregue esteja de acordo com as expectativas do cliente. Por isso, é importante observar que necessidades o sistema deverá atender, por meio do levantamento de requisitos. O projeto deve considerar todos os aspectos que envolvem o ciclo do desenvolvimento de sistemas, desde a elaboração e o desenvolvimento até a conclusão. Além disso, um projeto de sistemas web deve proporcionar viabilidade econômica, redução de erros e riscos, produtividade, segurança, entre outros pontos. Ao longo deste material, serão abordadas as principais tecnologias e tendências em desenvolvimento de sistemas web e uma seção específica sobre gestão de projetos de sistemas.

Tecnologias

Os sistemas web são executados diretamente no browser (ou navegador), não sendo necessário realizar uma instalação na máquina do usuário, como ocorre com os sistemas tradicionais. Para que seu funcionamento ocorra, o sistema depende de um servidor web, de solicitações realizadas pelos usuários, através de requisições emitidas no navegador pelo protocolo HTTP ou HTTPS e da resposta do protocolo. Ou seja, toda vez que um site é acessado no navegador de internet (cliente), é realizada uma requisição para um servidor web (servidor) através do protocolo HTTP ou HTTPS. Esse servidor recebe a requisição e retorna como resposta ao navegador do cliente. Essa resposta pode ser páginas, imagens, documentos, entre outros. Algo muito importante para o funcionamento dos sistemas web está relacionado com a comunicação do cliente com o servidor. Tecnologias para desenvolvimento web

Conheça, a seguir, os tipos das principais tecnologias usadas no desenvolvimento front-end e back-end.

Desenvolvimento front-end

É a parte de um site com a qual o usuário interage diretamente. Também é conhecido como o “lado do cliente” do aplicativo. Inclui tudo o que os usuários experimentam diretamente: cores e estilos de texto, imagens, gráficos e tabelas, botões, cores e menu de navegação. A estrutura, o design, o comportamento e o conteúdo de tudo que é visto nas telas do navegador, quando sites e aplicativos web ou móveis são abertos, são implementados por desenvolvedores de front-end. Cabe ao desenvolvedor garantir que o site seja responsivo, ou seja, que ele seja executado corretamente em dispositivos de todos os tamanhos, sem nenhuma parte se comportar de forma anormal, independentemente da resolução da tela. Veja as principais linguagens utilizadas a seguir.

HTML

Significa hypertext markup language. É usado para projetar a parte do front-end das páginas da web usando uma linguagem de marcação. HTML é a combinação de hipertexto e linguagem de marcação. O hipertexto define o link entre as páginas da web. A linguagem de marcação é usada para definir a documentação de texto dentro da tag que define a estrutura das páginas da web.

CSS

As folhas de estilo em cascata, chamadas de CSS – Cascading Style Sheets, são uma linguagem de design simples, que visa simplificar o processo de tornar as páginas da web apresentáveis. CSS permite que você aplique estilos a páginas da web.

JavaScript

É uma linguagem de script famosa, usada para criar mágica nos sites e torná-los interativos para o usuário. O JavaScript é usado para aprimorar a funcionalidade de um site, deixando-o mais rápido e fácil.

Existem muitas outras bibliotecas e framework para realizar o desenvolvimento front-end, por exemplo, Bootstrap, jQuery, Angular, React, Vue, Flutter e muito mais.

Desenvolvimento back-end

Responsável por fazer os sites executarem no lado do servidor. Ele armazena e organiza os dados e garante que tudo no lado do cliente do site funcione bem. Os desenvolvedores de back-end também criam serviços que processam a lógica de negócios e acessam outros recursos, como bancos de dados, servidores de arquivos, serviços em nuvem e muito mais. Java

É uma das linguagens e plataformas de programação mais amplamente utilizadas, devido a sua portabilidade, robustez e orientação a objetos. Extremamente popular, pois tem uma vasta gama de ferramentas e frameworks, como Spring Framework, desenvolvimento de API RESTful e arquitetura de microsserviços.

ASP.NET Core

É uma linguagem open-source, multiplataforma, criada pela Microsoft e pela comunidade. Extremamente leve, rápida e modular. Existem bibliotecas e frameworks que auxiliam no desenvolvimento da linguagem ASP.NET Core, por exemplo Entity Framework, Blazor Framework, entre outros.

O que é desenvolvimento full-stack?

Como desenvolvedor, você não precisa se contentar apenas com front-end ou back-end; você pode fazer os dois como desenvolvedor full-stack. Os desenvolvedores full-stack podem criar sites e aplicativos da web que renderizam no lado do cliente (front-end) e no lado do servidor (back-end).

PWA são aplicativos web que parecem aplicativos móveis nativos. Eles tiram proveito dos recursos dos navegadores de internet, oferecendo experiência de usuário semelhante a um aplicativo móvel. Funcionam off-line e têm capacidade de enviar notificações push, acesso a hardware, entre outros recursos. Exemplos de aplicações que utilizam: Twitter, Gmail, entre outros.

No passado, a maioria dos aplicativos da web eram desenvolvidos como aplicativos de várias páginas. Quando um usuário clicava em um link ou navegava para uma seção diferente do aplicativo da web, uma nova página era carregada. Mas o que é um aplicativo de página única? É um aplicativo da web que não precisa ser recarregado durante o uso. Em vez de carregar uma nova página, ele reescreve dinamicamente a página atual com novo conteúdo do servidor da web.

Nos aplicativos da web, as notificações push são usadas para fornecer informações aos usuários finais, por exemplo, uma nova postagem em um blog. Tudo isso é possível sem a necessidade de se registrar e deixar credenciais, como e-mail. Isso fornece canais adicionais para as empresas interagirem com seu público-alvo e oportunidades de marketing adicionais. Ferramentas para usar: PushCrew, ZoPush, entre outras.

API (application programming interface) significa interface de programação para aplicativos, sendo um intermediário que permite a comunicação entre diferentes softwares. O desenvolvimento de API possibilita integrar seu site ou serviço web com outros sistemas, como redes sociais e fornecedores de terceiros. À medida que seu site cresce, o uso de APIs facilita a comunicação com outras plataformas, dispositivos e conexões, de forma rápida e eficiente, tornando-o mais flexível e escalável.

Gestão de projetos

Anteriormente, foi definido o que são projetos, e gerenciar projetos significa orientar uma equipe com um enfoque organizacional, aplicando conhecimentos, habilidades e técnicas para o desenvolvimento de um software, no caso. Para isso, são utilizadas metodologias em desenvolvimento de software na gestão de projetos, que ajudam a estruturar, planejar e controlar o processo de desenvolvimento de software, assegurando um fluxo organizado e bem direcionado.

As metodologias clássicas são baseadas em etapas bem definidas, seguindo um modelo sequencial, em que a entrega do software ocorria somente na última etapa, como o método cascata. Já os métodos ágeis usam o desenvolvimento interativo e incremental como base. O software é construído de etapa a etapa, e novas funcionalidades são disponibilizadas de tempo em tempo. Além disso, os métodos ágeis têm um ponto de vista mais leve e centrado nas pessoas do que as metodologias clássicas. De forma geral, os processos ágeis usam a interação e o feedback contínuo em um sistema, e isso faz toda diferença para uma gestão de projetos. A seguir, veja os principais métodos ágeis empregados em gestão de projetos.

Scrum

É uma metodologia de gerenciamento de projetos com prazos de entrega curtos e em constante mudança, aplicando-se soluções incrementais para problemas complexos. Portanto, é fundamental que a equipe envolvida compreenda perfeitamente todos os processos do projeto. Para definir como a equipe desempenhará suas tarefas, entram em cena os conceitos de atores, reuniões, artefatos e papéis. Alguns elementos do Scrum são:

Scrum master

Responsável por garantir que o método está sendo bem aplicado. Deve conhecer a metodologia e assegurar que cada agente cumpra seu papel.

Product owner

Agente que definirá os requisitos para formar o product backlog, além de garantir que a equipe entenda os itens especificados.

Product backlog

É a lista de necessidades funcionais do software, em ordem de importância, alterada conforme as necessidades da equipe e do cliente.

Sprint

Ciclo de 30 dias em média, com tarefas finalizadas para entrega parcial do projeto. Reuniões diárias de 15 minutos avaliam seu desempenho e planejam as próximas 24 horas.

Equipe scrum

É composta pelo time de programadores e pelo cliente. Normalmente, essa equipe contém até sete pessoas.

Sprint backlog

Partindo do product backlog, a equipe Scrum seleciona o sprint backlog que será realizado durante uma fase de sprint. No final da sprint, uma parte já funcional do produto é entregue ao cliente, que não precisa esperar até que o produto final esteja pronto. Dessa forma, ele pode testar e já utilizar as funcionalidades que foram estabelecidas em uma ordem de prioridade.

Ilustração das etapas do processo Scrum, iniciando com a ideia/visão do projeto documentada no product backlog, seguida da etapa de planejamento da sprint, que, por meio do time de desenvolvimento, gerará o sprint backlog. A sprint do time de desenvolvimento leva, em média, duas semanas. Após finalizada, é feita uma revisão em que se atualiza o product backlog inicial para seguir o novo ciclo de desenvolvimento. Durante as sprints, existem reuniões diárias de até quinze minutos. Ao final de todos os ciclos do processo, tem-se o produto.

Kanban

É uma metodologia ágil que não é necessariamente iterativa. Processos como Scrum têm iterações curtas que imitam o ciclo de vida de um projeto em pequena escala, com início e fim distintos para cada iteração. O Kanban permite que o software seja desenvolvido em um grande ciclo de desenvolvimento. Apesar disso, o Kanban é um exemplo de metodologia ágil porque cumpre todos os princípios por trás do método ágil, pois, embora não seja iterativo, é incremental. O princípio por trás do Kanban, que permite que ele seja incremental e ágil, é o rendimento limitado. Sem iterações, um projeto Kanban não tem pontos de início ou término definidos para itens de trabalho individuais, cada um pode começar e terminar independentemente um do outro, e os itens de trabalho não têm duração predeterminada para esse assunto. Em vez disso, cada fase do ciclo de vida é reconhecida como tendo uma capacidade limitada de trabalho a qualquer momento.

Custos

O gerenciamento de custos de um projeto envolve aspectos de estimativa, determinação de orçamentos e controle de custos. O orçamento e o tamanho do projeto determinam os custos necessários para o gerenciamento de projetos. Todas as decisões relacionadas ao projeto devem ser regidas pelo orçamento estimado do projeto. Se os custos gerais do projeto não excederem o orçamento, você poderá concluir que o gerenciamento dos custos do projeto será bem-sucedido.

Calendário de tarefas

O calendário de tarefas é uma lista de atividades ou de etapas de um projeto com prazos definidos para início e conclusão, uma vez que os projetos têm fases iniciais de implantação, delegação de tarefas, metas e prazos. Os calendários podem ser incrementados com o controle e o encadeamento de várias tarefas, além do uso de software para monitoramento e compartilhamento entre usuários e equipes.

Exemplos de software

Relatório de acompanhamento

O relatório de acompanhamento é um documento que mostra o desempenho e o andamento do projeto, a fim de garantir que os principais problemas sejam descritos e as devidas atitudes sejam tomadas.

Além disso, o relatório de acompanhamento deve responder a quatro perguntas: “Onde estamos atualmente na execução do projeto? Quais são os próximos passos para o projeto? Quais são os obstáculos que temos que superar para entregar o projeto? Quais são as métricas-chave do projeto?”.

Para auxiliá-lo na compreensão deste relatório, serão apresentados os gráficos de Burndown, no método Scrum, que auxiliam como está a produtividade da sua equipe em relação ao desenvolvimento planejado com o real, ou seja, o executado até o momento.

Exemplo de um projeto de sistema web gerenciado no Kanban do Trello

Imagine um projeto de um sistema web para acompanhar o desenvolvimento de uma loja virtual, um e-commerce em que os clientes podem comprar seus produtos. A principal necessidade do solicitante desse projeto é vender seus produtos de forma eficiente. Para isso, é essencial que o sistema permita o cadastro dos produtos e exiba uma listagem completa, mostrando todos os itens com suas características e preços.

Além disso, o sistema precisa oferecer as principais formas de pagamento, como boletos bancários e cartões de crédito. Portanto, o sistema precisará se comunicar com serviços externos, como APIs de pagamento, para realizar transações financeiras de maneira segura e eficiente. Considerando esse exemplo do e-commerce, você pode dividir o projeto em tarefas menores e distribuí-las em categorias como “A fazer”, “Em Processo” e “Concluído”. Cada tarefa receberia um nível de esforço (em dias ou horas), e com isso é possível estimar a quantidade de trabalho a ser realizado dentro de uma sprint.

Exemplo do backlog de tarefas

Cadastro de produto: implementar a funcionalidade de novos produtos.

Esforço: 3 dias

Listagem de produtos: desenvolver a página que exibe todos os produtos cadastrados com suas características e preços.

Esforço: 2 dias

Integração com API de pagamentos: configurar a comunicação com uma API de pagamentos que permita transações com boletos e cartões de crédito.

Esforço: 4 dias

Veja, na imagem a seguir, as tarefas registradas no quadro backlog do Kanban no Trello antes de iniciar o desenvolvimento.

A imagem apresenta quatro quadros no Kanban do Trello. No primeiro quadro, há o nome de Backlog, com três tarefas registradas. Ao lado, há os quadros “A fazer”, “Em andamento” e “Concluído”. É possível acompanhar o desenvolvimento das tarefas quando elas forem movidas de quadro para quadro.

Conforme os desenvolvedores do software iniciam o desenvolvimento das tarefas, elas são movidas para o quadro “A fazer”, na sequência, avançam para “Em progresso”, e finalmente para “Concluído”. Isso permite uma visão clara do processo de desenvolvimento do projeto.

Encerramento

O sucesso de um projeto web vai além da simples escolha de tecnologias adequadas. A gestão de projetos desempenha um papel crucial nesse processo, garantindo que o controle de custos, prazos e relatórios de acompanhamento sejam executados de forma eficiente para que as metas estabelecidas sejam atingidas. Um gerenciamento de projetos eficaz, quando aliado a metodologias ágeis como o Scrum, possibilita a entrega de resultados de alta qualidade, respeitando tanto o orçamento quanto os prazos definidos. Esse modelo ágil promove adaptações rápidas e eficientes, permitindo que a equipe responda prontamente a mudanças e novos desafios que possam surgir ao longo do projeto.

Além disso, a integração de práticas ágeis na gestão de projetos web não apenas otimiza a execução das tarefas, mas também fortalece a comunicação e a colaboração entre as partes envolvidas. Portanto, o equilíbrio entre a escolha de tecnologias adequadas e uma gestão de projetos bem estruturada é essencial para o desenvolvimento de soluções web robustas, dentro dos parâmetros estabelecidos de tempo e custo.
