Domínios de internet: conceito, registro e disponibilidade de serviços

Para acessar um site na web, é necessário abrir o navegador e informar o endereço da URL desejada, em um processo semelhante ao pesquisar uma rua no GPS, informando seu nome para localizá-la. Entretanto, assim como as ruas têm um número chamado CEP para sua identificação, também existe uma sequência numérica para acessar um endereço na web, chamado endereço IP – Internet Protocol, porém, para facilitar sua memorização, utiliza-se seu nome de domínio. Neste conteúdo, você aprenderá o conceito desses domínios de internet, o Domain Name System, também conhecido como DNS. Além disso, verá como realizar o registro de um domínio e alguns serviços que são ofertados.

Conceito

O DNS é responsável por gerenciar e associar os nomes de domínio aos endereços IP. Por exemplo, ao digitar o endereço IP 172.217.29.67 na barra de endereços de um navegador, você será redirecionado ao site do Google. Confira na animação a seguir.

Animação da tela de um navegador, no qual é digitado o endereço IP número 172.217.29.67, na barra de endereço. Ao pressionar o botão “Enter”, é exibida a página inicial do site do Google.

Um nome de domínio é composto basicamente por duas partes: o nome e a extensão. O nome é escolhido pelo usuário, porém este corre o risco de realizar uma escolha que não está disponível, pois o domínio é uma forma de identificar uma infraestrutura de servidores na internet e disponibilizar acesso com flexibilidade aos serviços ofertados ao público, devendo este ser único, ou seja, não é possível que existam dois domínios com o mesmo nome disponíveis na internet. Já a extensão vai indicar a natureza do site. As extensões mais conhecidas são “.com”, “.org” e/ou “.gov”, com as seguintes classificações, respectivamente: comercial, organização sem fins lucrativos e governos.

Além de facilitar a memorização para o acesso a um site, um domínio é útil também em outros aspectos, como credibilidade, profissionalismo e endereços de e-mail personalizados com o domínio escolhido, comunicando, de modo eficiente, o marketing de uma empresa. Por fim, pode ser útil para SEO (Search Engine Optimization), pois, com palavras-chaves adequadas, o site pode subir no ranqueamento de pesquisa de um serviço de busca. Veja diferentes níveis de domínio a seguir.

É a parte do domínio que expressa que o site é de determinado país, composto pelas duas letras após o TLD. No Brasil, é usado o código “.br”.

É semelhante a um TLD, no entanto, não precisa da sigla de identificação do país. É voltado a situações especificas, como, por exemplo, o “.edu”, pertencente a instituições educacionais, ou “.mil”, para uso militar.

Subdomínios são vantajosos porque permitem aos administradores de site criarem seções distintas sem adquirir um novo domínio. Em vez disso, eles podem criar subdomínios direcionando-os a diretórios específicos no servidor.

DNS

É um serviço que realiza a tradução do endereço IP para a nomenclatura dada ao domínio, conforme visto. O DNS autoritativo funciona como fonte oficial de informações sobre um domínio, com duas funções principais. Conheça-as a seguir.

É chamado de “autoritativo” porque é a última palavra sobre o domínio. Outros servidores DNS, que atuam como “intermediários”, pedem as informações a ele para fornecer ao usuário final o caminho certo. O “recursivo” é outro serviço DNS conhecido, que possibilita o acesso ao endereço informado, caso já esteja em cache, no entanto, ao não encontrar informações em cache, é realizada uma requisição ao DNS autoritativo. Dessa forma, o DNS recursivo visa reduzir o tempo de acesso ao endereço buscado.

Funcionamento do DNS

O DNS funciona da seguinte maneira: o usuário abre o seu navegador e informa o endereço que deseja acessar. Ao confirmar o endereço, é enviada uma requisição para o resolvedor DNS através do provedor de serviços contratado. Ao receber a requisição, é feito um novo encaminhamento para um dos servidores de serviço TLD. O servidor, por sua vez, responde com a informação do nome associado ao domínio solicitado. Essa informação é enviada do resolvedor DNS para o usuário que solicitou o acesso com o endereço IP solicitado. Em posse do endereço IP, o navegador envia nova requisição a ele, que retorna com a página web para o navegador mostrar a página HTML.

Ilustração do fluxo de operação DNS. O ícone do resolvedor DNS está conectado por uma seta ao ícone de um servidor DNS, com o número dois indicando o segundo passo. O fluxo mostra o resolvedor DNS realizando uma busca pelo endereço IP do domínio. O servidor DNS está conectado ao resolvedor DNS com uma seta retornando o IP do domínio, indicado pelo número três, representando o terceiro passo. O resolvedor DNS envia o endereço IP de volta ao navegador, indicado pelo número quatro e uma seta apontando do resolvedor DNS para o navegador, completando a entrega do IP. O navegador, agora em posse do endereço IP, envia uma solicitação de página ao servidor de hospedagem, indicada pelo número cinco e uma seta apontada para o servidor de hospedagem. O servidor de hospedagem responde ao navegador com a página HTML solicitada, ilustrado pelo número seis, com uma seta do servidor de hospedagem ao navegador, fechando o ciclo.

Registro

Existe um processo de registro que precisa ser seguido e respeitado, afinal, como já visto, os domínios precisam ser únicos para que possam funcionar corretamente, assim como alguns TLDs necessitam seguir regras especificas. Os domínios devem ter de dois a 63 caracteres, é possível utilizar letras, números e hifens, no entanto, não é possível iniciar ou concluir o nome com o hífen. Um domínio pode ser reservado ou comprado, utilizando serviços de registros, ofertados por empresas de hospedagem de sites ou diretamente por meio de registradores. No Brasil, existe o Registro.br, disponibilizado pelo Nic.br, para encontrar esses serviços, é recomendado pesquisá-los em um buscador, como o Google.

Disponibilidade de serviços

Ao efetuar o registro do domínio, como já referenciado, é necessário procurar um serviço de hospedagem, podendo ser serviços gratuitos e/ou pagos. Eles são essenciais para que o site possa ser hospedado e fique disponível na internet para os demais usuários acessarem. Outro tipo de serviço para uso é a possibilidade de criação de contas de e-mail personalizadas. Também é possível utilizar a tecnologia de certificados digitais SSL (secure socket layer, um protocolo de comunicação de dados), que criptografam os dados nas requisições e nas respostas entre a máquina do usuário e o servidor do site, implementando, assim, a credibilidade do site, pois garante a privacidade dos dados. É necessário buscar o certificado com fornecedores de certificados digitais. Para estabelecer uma conexão SSL, o servidor web precisa ter um certificado SSL, no qual são preenchidas diversas informações sobre a identidade do servidor, site e empresa. Devido à sua praticidade, em muitos casos, o certificado digital substitui a operação de login e senha. Durante esse processo, o servidor gera duas chaves criptográficas: uma pública e outra privada. Elas são associadas a diferentes certificados. A chave pública é incluída no CSR (certificate signing request), um arquivo que contém detalhes sobre o servidor. Como a chave pública não precisa ser confidencial, a certificadora valida as informações contidas no CSR e, em seguida, emite o certificado SSL, que contém esses dados. Com isso, o servidor consegue estabelecer a conexão criptografada utilizando o certificado SSL e a chave privada. Esse procedimento acontece de forma que os usuários só percebem a conexão segura por meio de um ícone de cadeado verde, geralmente exibido no canto esquerdo do navegador. No entanto, o Google Chrome, a partir da versão 117, em setembro de 2023, substituiu por um ícone de ajustes. Ao clicar sobre esse ícone, é possível visualizar e verificar os detalhes do certificado SSL.

Tela do site do Google mostrando a barra de endereços do navegador Google Chrome com o endereço do site “google.com.br”, abaixo há uma caixa com sua URL, um ícone de cadeado com o texto “A conexão é segura”, um ícone de cookies com o texto “Cookies e dados do site” e um ícone de engrenagem com o texto “Configurações do site”.

É possível gerar um certificado digital, contratando o serviço de uma empresa que tenha autoridade certificadora. Por fim, também é possível se valer do serviço de privacidade WHOIS, um protocolo para consultar informações sobre registros de domínios, ofertado pelo registrador, no qual os dados do cliente que realizou o domínio são mantidos em privacidade, mantendo o anonimato e evitando, por exemplo, o envio de spams.
